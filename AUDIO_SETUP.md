# Audio Setup Guide

The app uses a three-tier audio system:

1. **Pre-generated MP3 files** (best quality, consistent for everyone) — generated by `scripts/generate-audio.cjs`
2. **Browser TTS fallback** — works on devices that have the language voice installed
3. **Silent + hidden audio button** — final fallback so the app still works

This guide walks you through setting up Tier 1 — the real audio files.

---

## Why this matters

Browser text-to-speech is **device-dependent**. iPhones tend to have most languages, Android phones often don't, budget devices have almost none. Without pre-generated audio, your TikTok visitors on Android phones get silent buttons.

Pre-generated MP3s solve this: every user, every device gets the same audio, served from your Vercel deployment as static files.

## What you'll spend

**$0** if you stay in Google's free tier, which covers far more than your current vocab needs.

- Google Cloud TTS free tier: 4 million characters/month for standard voices, 1 million for premium WaveNet voices.
- Your full vocab (~462 words across 8 languages) = under 5,000 characters.
- You'd need about **200 launches per month** to even approach the free tier limit.

Google requires a credit card for verification but will not charge you while you stay under the free tier.

## Step 1: Set up Google Cloud (15 minutes, one-time)

1. Go to **https://console.cloud.google.com**
2. Sign in with a Google account (use a fresh one if you want to keep this isolated)
3. Click **Create Project** at the top — name it something like "lingua-tts"
4. Once the project is created, click into it
5. In the left sidebar, find **APIs & Services → Library**
6. Search for **Cloud Text-to-Speech API** — click it, then click **Enable**
7. It'll prompt you to set up billing (credit card). Add a card. **Free tier still applies — you won't be charged unless you exceed it.**

## Step 2: Create credentials

1. In the left sidebar: **APIs & Services → Credentials**
2. Click **+ Create Credentials** at the top → **Service Account**
3. Name it `lingua-audio-generator` — leave permissions as default — Done
4. Click on the service account you just created
5. Click the **Keys** tab → **Add Key** → **Create New Key** → **JSON** format → **Create**
6. A `.json` file downloads to your computer

## Step 3: Put the key in your project

1. Move the downloaded JSON file into your `lingua-pro` project folder
2. **Rename it** to `google-tts-key.json` (exact name)
3. **Verify .gitignore is protecting it:** open `.gitignore`, confirm `google-tts-key.json` is in the list. **Never commit this file** — it grants full access to your Google Cloud account.

## Step 4: Install the Google library and run

In Terminal, from your project folder:

```bash
npm install @google-cloud/text-to-speech
npm run audio
```

The script will run for a few minutes (it generates ~460 audio files at a polite rate). You'll see output like:

```
=== UR ===
  ✓ ur/ur_0001 سلام
  ✓ ur/ur_0002 شکریہ
  ...
```

When it finishes, you'll have a `public/audio/` folder containing every word as an MP3.

## Step 5: Push to GitHub and redeploy

```bash
git add public/audio
git commit -m "Generate audio files for all vocab"
git push
```

Vercel will automatically pick up the new files. They're served as static assets at:

```
https://your-vercel-domain.vercel.app/audio/{lang}/{wordId}.mp3
```

The frontend already knows where to look. From the moment Vercel finishes redeploying, every user on every device gets the same audio.

---

## Re-generating audio later

If you add new words to your vocab JSON files, just run `npm run audio` again. The script **skips files that already exist**, so it only generates what's new. Fast and cheap.

If you want to **re-record everything with a new voice**, delete the `public/audio/` folder and run `npm run audio` again.

---

## Switching voices

Edit `scripts/generate-audio.cjs`. The `VOICE_CONFIG` object at the top maps each language code to a voice. To browse all available voices:

```bash
gcloud auth application-default login
gcloud ml voice list-voices --language-codes ur-PK
```

Or visit **https://cloud.google.com/text-to-speech/docs/voices** for the full list.

---

## What the user sees

- **First time loading the app**: their browser fetches the MP3s from Vercel (small files, ~5KB each, so it's instant)
- **Tap the 🔊 button**: instant playback of the high-quality MP3
- **Offline / mp3 doesn't load**: silently falls back to browser TTS
- **Browser TTS also unavailable**: the audio button is hidden so they're not confused by a silent button
